**웹 프로그래밍**

서버 사이드 스크립트 코드 작성   

**서버 사이드 스크립트 언어 - JSP**

브라우저에서 요청한 HTML 문서에 서버 사이드 스크립트 언어가 포함되어 있으면, 서버는 이 부분을 자신이 처리한 후 그 HTML 문서를 브라우저에 보내줍니다.

**서블릿**

클라이언트의 요청을 처리하고 응답을 생성하는 자바 클래스입니다.

따라서 원하는 작업을 수행하려면 서블릿 클래스를 수정하거나 새로운 서블릿 클래스를 작성해야 합니다.

**서블릿 컨테이너, =WAS, =아파치 톰캣**

서블릿을 관리하고 실행하는 역할을 합니다. 

**JSP**

JSP 파일을 작성하면, 웹 서버는 이를 서블릿으로 컴파일하여 실행합니다. 따라서 JSP를 사용하면 서블릿 클래스를 직접 작성하지 않고도 동적인 웹 페이지를 쉽게 만들 수 있습니다.

서블릿이 자바 소스 코드속에 HTML 코드를 넣어 작성한다면, JSP는 이와 반대로 HTML에 자바 코드를 넣는 구조이다.

### JSP 구성요소 7

1. 디렉티브(directive)
2. 스크립트
   - 스크립틀릿(scriptlet)
   - 표현식(expression)
   - 선언(declaration)
3. 표현 언어(expression language)
4. 기본 객체(implicit object)
5. 정적인 데이터
6. 표준 액션 태그(action tag)
7. 커스텀 태그(custom tag)와 태그 라이브러리(JSTL)

### 1. 디렉티브

JSP 페이지의 최상단에 선언하며 페이지 처리 방법 설정

```jsp
<%@ page | include | taglib 속성="값" %>
```

### 2. 스크립트

JSP 페이지에서 자바 코드 사용

**선언문**

```jsp
<%! 변수/함수 정의 %>
```

**스크립틀릿**

```jsp
<% 조건문/반복문 %>
```

**표현식**   

`out.print(출력할 내용)`과 같음

```jsp
<%= 출력할 내용 %>
```

### 3. 표현 언어(EL)

표현 언어를 사용하면 굳이 스크립트를 쓸 필요 없음   
스크립트와 마찬가지로 기본 객체 호출 가능

### 4. 기본 객체(EL)

### 5. 정적인 데이터(HTML, CSS, JS, 이미지 리소스 등)

### 6. 표준 액션 태그

스크립트, 표현 언어가 많아질수록 JSP코드는 복잡해진다.   
자바로 구현해야 되는 기능을 JSP에서 제공하는 태그로 구현한다.

```jsp
<jsp:forward | include 속성="값">
  <jsp:param 속성="값">
</jsp:forward | include>
```

```jsp
<jsp:useBean 속성="값" />
<jsp:setProperty 속성="값" />
<jsp:getProperty 속성="값" />
```

### 7. 커스텀 태그 & JSTL

표준 액션 태그에 없는 태그를 만들 수 있다.
JSTL는 커스텀 태그를 모아놓은 라이브러리이다.

사용하려면 taglib 디렉티브 태그를 선언해주어야 한다.

```jsp
<%@ taglib prefix="my" tagdir="/WEB-INF/tags/myTags/" %>

<my:tagFile01 속성="값"/>
```

### 상태 코드

서버에서 클라이언트에게 알려주는 요청 결과

|상태 코드|설명|
|---|---|
|200|정상 처리|
|307|임시로 리다이렉트(response.sendRedirect() 이용 시)|
|400|요청이 잘못됨(해석 불가)|
|403|요청을 거부|
|404|자원이 존재하지 않음|
|405|요청한 메서드(GET, POST 등)을 거부|
|500|서버 내부 에러(JSP 예외 발생)|
|503|일시적으로 서비스 제공 불가|
